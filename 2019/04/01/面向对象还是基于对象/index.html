面向对象还是基于对象

2019.04.01 Joy917  热度 ℃

笔者第一次了解「面向对象」这个概念，还是在大学学习 Java 的时候。耳熟能详的「面向对象三大特性」那也是张口就来：封装、继承、多态。但是最近在学习「重学前端」课程的时候，文中提到了这个问题，我一时竟有些语塞。

想要区分二者的关系，我们还是要回到对「对象」这个概念的理解。

对象（Object）由于翻译的原因，中文的语境不太容易理解。其实在英文中，Object 指一切事物的总称，更能反映面向对象编程（oop）的抽象思维。

参考《面向对象分析与设计》，对象的三个本质特征分别是：

  • 具有唯一标志性：即使完全相同的两个对象，也并非同一个对象。
  • 有状态：同一个对象可能处于不同的状态之下。
  • 具有行为：对象的状态可能会随着行为而改变。

而在 Java 中，后面两个特征分别称为属性、方法。

而面向对象编程，是一种仿现实世界的高度抽象的编程；而基于对象，只是语言设计时的基础设施，由对象提供而已。并不是只有「类」的概念的语言才称之为面向对象，关键在于区分它到底有没有去实现对象的本质特征。例如 JS，虽然它是一种高度动态的对象系统（可以运行时改变对象的行为，Java做不到），但是仍然属于面向对象。

其实仔细想一想，我们就能从语言的设计思想上，更宏观的去回答这个问题。
